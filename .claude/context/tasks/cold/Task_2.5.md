# Task 2.5: Food Truck Profile Backend

## Objective
Implement the backend APIs for food truck profile management including menu, photos, and pricing.

## Requirements (from PRD)
- F4: Food Truck Profiles
- Profile data: truck name, cuisine type, menu with pricing, photos, capacity, event type specializations
- Menu management with items and pricing
- Photo gallery management
- Cuisine types and event specializations

## Acceptance Criteria
- [ ] GET /api/vendors/[id] - Public profile view
- [ ] PUT /api/vendors/[id] - Update vendor profile
- [ ] POST /api/vendors/[id]/menu - Update menu items
- [ ] POST /api/vendors/[id]/photos - Upload photos
- [ ] DELETE /api/vendors/[id]/photos/[photoId]
- [ ] GET /api/vendors/[id]/stats - Vendor statistics
- [ ] Cuisine type enum support
- [ ] Event type enum support
- [ ] Capacity range validation
