## Lessons Learned

1. **Structure matters**: Clear hierarchy (README → Category → Guide) helps users navigate
2. **Cross-linking**: Internal links between related articles improve discoverability
3. **Dual audience**: Balance customer and vendor needs in knowledge base
4. **Searchability**: Use consistent terminology and keywords throughout
5. **Completeness**: Cover edge cases and troubleshooting, not just happy path
6. **Tone**: Professional yet friendly, accessible yet comprehensive
7. **Examples**: Real-world examples make abstract concepts concrete
8. **Visual planning**: Even without images, describe where they would go

---

## Knowledge Base Best Practices

1. **Organization**: Group by user type (customer/vendor) and topic
2. **Navigation**: Multiple entry points (README, FAQ, topic pages)
3. **Search optimization**: Use headers, bullet points, consistent terms
4. **Maintenance**: Mark last updated dates for freshness
5. **Coverage**: FAQ, how-to guides, troubleshooting, contact support
6. **Accessibility**: Plain language, clear formatting, scannable content
7. **Cross-reference**: Link related articles for deeper exploration
8. **Progressive disclosure**: Start simple (FAQ) → detailed (how-to) → deep (troubleshooting)

## Reusable Templates

### How-To Guide Structure
1. Overview/Purpose
2. Prerequisites/Requirements
3. Step-by-step instructions
4. Troubleshooting section
5. Next steps/related guides
6. Support contact

### FAQ Structure
1. Group by topic/category
2. Bold questions, regular text answers
3. Link to detailed guides where applicable
4. Keep answers concise (50-200 words)
5. Use examples for clarity

### Troubleshooting Structure
1. Symptoms description
2. Common causes
3. Numbered solutions
4. "Still having issues?" → contact support
5. Prevention tips

## Project Context Learned

**Fleet Feast Platform Details**:
- Food truck marketplace (customers book vendors for events)
- Payment: 50% deposit (on acceptance) + 50% final (72h before event)
- Cancellation: 7+ days (100%), 3-6 days (50%), <3 days (0%)
- Vendor fees: 15% standard, 10% loyalty (50+ bookings)
- Response time: Vendors have 48 hours to accept/decline
- Payout: 7 days after event completion
- Stripe payment processing
- Service fee: 5-8% to customers

**User Types**:
- Customers: Book food trucks for events
- Vendors: Food truck operators seeking bookings
- Platform: Facilitates connection and payments

**Key Flows**:
- Customer: Search → Book → Pay → Event → Review
- Vendor: Apply → Setup Profile → Receive Requests → Accept → Serve → Get Paid
